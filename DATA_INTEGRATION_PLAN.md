# 한투 KIS + DART 데이터 통합 개발 계획

## 1. 개발 목표

현재 앱은 기술·거래량·수급·실적 성장·추정 주가 괴리율을 바탕으로 현재 주가를 평가합니다.

다음 단계 목표는 아래와 같습니다.

```txt
1. 가능한 데이터는 최대한 한투 KIS API에서 자동 수집
2. DART는 공시·재무제표 원천 확인과 주요 공시 안내에 사용
3. 자동 데이터가 부족한 항목은 수동 입력으로 보완
4. 모든 데이터 출처를 화면에 표시
5. 추정 주가와 종합 신뢰도의 신뢰도를 높임
```

---

## 2. 데이터 역할 분리

### 2-1. 한투 KIS API 우선 활용

한투 KIS API는 앱의 실시간·정량 데이터 중심으로 사용합니다.

```txt
현재가
기간별 시세
거래량·거래대금
외국인·기관 수급
재무비율
성장성비율
수익성비율
안정성비율
공매도
신용잔고
```

앱 반영 위치:

```txt
기술 점수
거래량·거래대금 점수
수급 점수
실적 성장 점수
추정 주가 참고 범위
추정 주가 신뢰도
위험 보정
```

---

### 2-2. DART 보완 활용

DART는 공시와 원천 재무 확인용으로 사용합니다.

```txt
최근 공시 목록
사업보고서
반기보고서
분기보고서
단일회사 재무제표
주요계정
주요 재무지표
배당 관련 공시
증자·감자 관련 공시
최대주주 변경
자기주식 취득/처분
```

앱 반영 위치:

```txt
공시·재무 자동 확인 섹션
실적 성장 데이터 검증
추정 주가 신뢰도 보정
공시 리스크 안내
데이터 출처 표시
```

## 3. 화면 추가 구조

새 섹션명:

```txt
공시·재무 자동 확인
```

화면 예시:

```txt
공시·재무 자동 확인

데이터 출처
- 한투 자동: 재무비율, 성장성비율, 수급
- DART 보완: 최근 공시, 주요계정
- 수동 입력: 컨센서스 추정 주가

최근 공시
- 2026.05.10 분기보고서 제출
- 2026.05.08 단일판매·공급계약 체결
- 2026.05.03 자기주식 취득 결정

재무 원천 확인
- 최근 보고서: 2026년 1분기보고서
- 매출액: 데이터 확인
- 영업이익: 데이터 확인
- 당기순이익: 데이터 확인

분석 반영 상태
- 한투 성장성비율 반영
- DART 주요계정 보완 확인
- 공시 특이사항: 없음 / 주의 필요
```

## 4. API Route 설계

### 4-1. 한투 API

현재 앱에 이미 있는 구조를 확장합니다.

```txt
/api/kis/fundamentals
```

확장 목표:

```txt
재무비율
성장성비율
수익성비율
안정성비율
```

추가 후보:

```txt
/api/kis/investors
/api/kis/short-selling
/api/kis/credit-balance
/api/kis/financial-ratios
```

우선은 파일을 많이 늘리지 않고 기존 `/api/kis/fundamentals`에 확장하는 방식을 추천합니다.

---

### 4-2. DART API

신규 route 후보:

```txt
/api/dart/disclosures
/api/dart/financials
/api/dart/company
```

우선순위는 아래 순서입니다.

```txt
1. /api/dart/disclosures
   - 최근 공시 목록 조회
   - 화면에 최근 5~10건 표시

2. /api/dart/financials
   - 단일회사 재무제표 또는 주요계정 조회
   - 매출액, 영업이익, 당기순이익 확인

3. /api/dart/company
   - 기업개황, corp_code 확인
   - 종목코드와 DART corp_code 매핑 보완
```

## 5. 데이터 출처 우선순위

실적 성장 분석의 우선순위는 아래처럼 둡니다.

```txt
1순위: 한투 자동 데이터
2순위: DART 보완 데이터
3순위: 저장된 수동 입력
4순위: 데이터 없음
```

다만 사용자가 수동 적용을 선택한 경우에는 수동 입력을 우선합니다.

```txt
자동 적용
→ 한투 자동 > DART 보완 > 수동 입력

수동 적용
→ 수동 입력 우선
```

## 6. 추정 주가 산정 반영

추정 주가는 다음 구조로 고도화합니다.

```txt
추정 주가 =
기술 추정 주가
+ 밸류에이션 추정 주가
+ 수급 보정
+ 거래대금 보정
+ 실적 성장 보정
+ 공시 리스크 보정
+ 공매도/신용 리스크 보정
```

### 한투 반영

```txt
성장성비율 긍정
→ 실적 성장 점수 가산
→ 추정 주가 신뢰도 가산

외국인·기관 동시 순매수
→ 수급 점수 가산
→ 추정 주가 신뢰도 가산

공매도 증가
→ 위험 보정 감점

신용잔고 급증
→ 위험 보정 감점
```

### DART 반영

```txt
최근 호재성 공시
→ 공시 참고 문구 추가
→ 신뢰도 보조 가산 검토

불확실성 공시
→ 공시 리스크 주의 표시
→ 신뢰도 감점 검토

최근 분기보고서 확인
→ 실적 데이터 신뢰도 가산
```

## 7. 개발 순서

### 1단계. 데이터 통합 계획 파일 추가

현재 문서입니다.

```txt
DATA_INTEGRATION_PLAN.md
```

목적:

```txt
앞으로 한투/DART 연동 기준을 고정
데이터 역할 혼선 방지
화면 반영 위치 정리
```

---

### 2단계. 한투 fundamentals 확장

수정 예상 파일:

```txt
app/api/kis/fundamentals/route.ts
lib/earningsGrowth.ts
components/analysis/EarningsGrowthSection.tsx
```

목표:

```txt
한투 재무·성장성 데이터가 있으면 실적 성장 점수 자동 계산
자동 데이터 출처를 화면에 표시
```

---

### 3단계. DART 최근 공시 route 추가

신규 예상 파일:

```txt
app/api/dart/disclosures/route.ts
lib/dart.ts
components/analysis/DisclosureSection.tsx
```

목표:

```txt
최근 공시 5~10건 표시
공시일, 공시 제목, 보고서 링크 표시
공시 특이사항 안내
```

---

### 4단계. 공시·재무 자동 확인 섹션 추가

수정 예상 파일:

```txt
app/page.tsx
components/analysis/DisclosureSection.tsx
```

목표:

```txt
한투 자동 데이터
DART 보완 데이터
수동 입력 데이터
각 출처와 반영 상태를 한 화면에 표시
```

---

### 5단계. 추정 주가 신뢰도 점수 추가

수정 예상 파일:

```txt
lib/score.ts
types/stock.ts
components/analysis/CompositeScoreSection.tsx
```

목표:

```txt
추정 주가 자체의 신뢰도를 별도 점수화
수급·실적·공시·위험 요소를 반영
```

## 8. 다음 작업으로 바로 진행할 항목

우선순위 1번:

```txt
DART 최근 공시 안내 섹션 추가
```

이유:

```txt
1. 공시 정보는 투자 판단 참고성이 큼
2. 추정 주가 신뢰도와 연결 가능
3. 화면에 바로 보이는 기능이라 사용성이 높음
4. DART API 키가 준비되면 바로 자동화 가능
```

우선순위 2번:

```txt
한투 fundamentals 확장
```

이유:

```txt
1. 실적 성장 자동화에 직접 연결
2. 수동 입력 의존도를 줄임
3. 추정 주가 산정 정확도 향상
```
